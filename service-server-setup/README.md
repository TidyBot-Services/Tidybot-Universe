# Service Server Setup

Automated discovery and documentation of ML services deployed on remote GPU servers.

## What it does

- Scans a remote server's service directory via SSH
- Detects services by looking for `server.py`, `Dockerfile`, `service.yaml`, etc.
- Extracts API endpoints from FastAPI route decorators
- Detects ports from argparse defaults, uvicorn configs, or Dockerfiles
- Checks which services are currently running (port listeners)
- Hosts a live catalog on localhost that skill agents can query
- Polls every 30 seconds for new or changed services
- Alerts when the remote server becomes unreachable

## Prerequisites

- SSH access to the remote GPU server (key-based auth)
- Python 3.10+ with `fastapi`, `uvicorn`, `pyyaml`

## Setup

```bash
cd ~/tidybot_uni/Tidybot-Universe/service-server-setup

# Install dependencies
pip install -r requirements.txt

# Run interactive setup (tests SSH, writes config.json)
bash setup.sh --server-ip YOUR_SERVER_IP --username YOUR_USER \
              --service-dir /path/to/services
```

## Usage

```bash
# Start the doc server
python3 service_scanner.py

# Or scan without starting the server
python3 service_scanner.py --scan-once
```

Then query:

```bash
curl http://localhost:8090/services         # list all services
curl http://localhost:8090/services/graspgen_server  # service details
curl http://localhost:8090/health            # server status
```

Or open http://localhost:8090/docs/html in a browser.

## API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/health` | GET | Catalog health + server connectivity |
| `/services` | GET | List all services with status |
| `/services/{name}` | GET | Detailed service info |
| `/services/{name}/endpoints` | GET | API endpoints for a service |
| `/docs/html` | GET | HTML documentation page |
| `/rescan` | POST | Trigger immediate rescan |

## Configuration

`config.json` (generated by `setup.sh`):

```json
{
    "server_ip": "YOUR_SERVER_IP",
    "username": "YOUR_USER",
    "service_dir": "/path/to/services",
    "local_port": 8090,
    "poll_interval_seconds": 30
}
```
